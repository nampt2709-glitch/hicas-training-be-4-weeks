================================================================================
  FILE: LogAnalyzer/ResultWriter.cs
  MỤC ĐÍCH: Ghi báo cáo kết quả benchmark ra file .txt
================================================================================

TẠI SAO CÓ FILE NÀY?
  Sau khi BenchmarkRunner chạy xong 6 lần thử, ta có BenchmarkReport gồm:
  - Chế độ (Word hay Error).
  - Đường dẫn file nguồn.
  - 6 dòng kết quả: mỗi dòng có nhãn (ví dụ "Sync + Sequential") và thời gian (ms).
  - TopItems: các từ/lỗi đã đếm (để in top 50).

  ResultWriter biến dữ liệu trong bộ nhớ thành file văn bản để mở bằng Notepad.


--------------------------------------------------------------------------------
CLASS: ResultWriter (static)
--------------------------------------------------------------------------------

  PHƯƠNG THỨC: Write(BenchmarkReport report)
  - Đầu vào: report (kết quả một lần chạy benchmark).
  - Đầu ra: đường dẫn đầy đủ tới file .txt vừa ghi.

  CÁC KHỐI LOGIC:

  (1) Thư mục Results cạnh thư mục chạy chương trình (AppContext.BaseDirectory).
      Tạo thư mục nếu chưa có.

  (2) Tên file theo thời gian: Result_yyyyMMdd_HHmmss_fff.txt
      Mỗi lần chạy không ghi đè file cũ.

  (3) StringBuilder để ghép nội dung (hiệu hơn nối chuỗi nhiều lần).

  (4) Phần đầu: Mode, tên file nguồn, thời điểm tạo báo cáo.

  (5) Phần PERFORMANCE: bảng hai cột - nhãn cách chạy và Time(ms) từ report.Runs.

  (6) Phần TOP 50:
      - Không có mục nào: in "No words..." hoặc "No error...".
      - Có: in TOP 50 WORDS hoặc TOP 50 ERRORS, lấy tối đa 50 phần tử đầu TopItems.

  (7) File.WriteAllText(..., Encoding.UTF8) để tiếng Việt hiển thị đúng.

LIÊN KẾT
  Program gọi BenchmarkRunner rồi ResultWriter.Write(report).
  FrequencyItem, BenchmarkRun, BenchmarkReport định nghĩa trong Counter.cs.
